Pools and Lanes:

Pools: In BPMN, a Pool represents a whole process or a participant in a process. It's the highest-level element in a BPMN diagram and often represents an entire organization, department, or system. A pool can contain multiple lanes and elements but is primarily used to delineate different organizations or major participants in a process.

Lanes: Lanes are subdivisions within a Pool. They are used to organize and categorize activities within a Pool. Lanes often represent different roles, departments, or types of activities within the organization. Each lane can contain various BPMN elements like tasks, events, and gateways, but they all pertain to the role or department the lane represents.

Examples:

1. Example 1: Customer Service Process
   - Pool: "Customer Service Department"
   - Lanes: 
     - "Support Team": Handling customer inquiries and issues.
     - "Technical Team": Dealing with technical problems reported by customers.
     - "Follow-up Team": Managing customer feedback and follow-up activities.

2. Example 2: Order Processing System
   - Pool: "Order Processing"
   - Lanes: 
     - "Sales Department": Taking customer orders and processing payments.
     - "Warehouse": Managing inventory and preparing orders for shipment.
     - "Logistics": Coordinating the delivery of orders to customers.

3. Example 3: Employee Onboarding
   - Pool: "HR Onboarding Process"
   - Lanes: 
     - "HR Department": Conducting initial employee orientation and paperwork.
     - "IT Department": Providing technical setup like email accounts and hardware.
     - "Team Leads": Introducing new hires to their respective departmental teams and responsibilities.

Gateways:

1. Exclusive Gateway (XOR): It is used to create alternative paths within a Process flow. For a given instance of the Process, only one of the paths can be taken. 
   Examples:
   - Order Processing: If an ordered item is in stock, the process moves to shipping; if not, it goes to backorder processing.
   - Loan Approval: Based on a credit score check, a loan application is either approved (proceed to loan processing) or rejected.
   - Travel Request: If the travel budget is within limits, the request is approved; otherwise, it's sent for further review.

2. Parallel Gateway (AND): It is used to split the process flow into multiple parallel paths or to converge parallel paths.
   Examples:
   - Employee Onboarding: New employees must complete several tasks simultaneously, like training, document submission, and workstation setup.
   - Event Planning: Catering, venue setup, and invitation distribution happen in parallel.
   - Product Launch: Concurrent activities like marketing, inventory stocking, and staff training are initiated.

3. Inclusive Gateway (OR): This gateway allows for one or more paths to be taken out of the available ones based on conditions. It's like a combination of Exclusive and Parallel Gateways.
   Examples:
   - Order Customization: Based on customer preferences, an order can include packaging, personalization, or both.
   - Incident Response: Depending on the incident's nature, notify management, alert security, or both.
   - Healthcare Check-up: Based on symptoms, a patient might need to undergo one or more of several tests.
   - Snack preparing: You can prepare one, two, or three snack types for a party.

4. Event-Based Gateway: This leads to a path based on the occurrence of an event, rather than on the evaluation of expressions like the others. 
   Examples:
   - Support Ticket Handling: The next step depends on whether the customer replies to the email or calls directly.
   - Auction System: The flow depends on whether a new bid is placed or the auction time expires.
   - Weather-Dependent Activities: Outdoor activities depend on whether the forecast is sunny or rainy.

5. Complex Gateway: It allows for more complex synchronization or merging behaviors that can't be captured by other gateways. Used less frequently.
   Examples:
   - Project Approval: A project moves forward only if a predefined combination of departmental approvals is received.
   - Manufacturing Process: The next phase of production starts only after multiple concurrent quality checks pass.
   - Investment Decision: Investments are made based on a complex set of criteria involving market conditions, risk assessments, and portfolio balance.

6. Exclusive Event-Based Gateway: A variation of the Event-Based Gateway, it allows for deciding the path based on which event occurs first.
   Examples:
   - Flight Rebooking: Passengers are rebooked either when they request it or automatically after a flight cancellation.
   - Online Auction: The auction ends when either a buyout price is met or the bidding period ends.
   - Emergency Response: Action is taken based on the first received alert, either from an automated system or a manual distress call.

Tasks:

1. Task: This is a basic unit of work in a process. It's a generic term that refers to a piece of work that isn't broken down into a finer level of business process detail.
    Examples:
    - Processing an Order: In a retail company, processing a customer's order from receipt to fulfillment.
    - Updating a Database: In a database management scenario, where a record needs to be updated.
    - Conducting a Performance Review: In an HR context, where an employee's performance is evaluated.

2. User Task: A task that is performed by a human actor. This typically involves human judgment and is often performed with the assistance of a software application.
    Examples:
    - Approving a Loan Application: A bank employee evaluates and approves or rejects loan applications.
    - Filling Out a Survey: A customer service representative fills out a feedback survey after a call.
    - Reviewing a Document: A legal professional reviews and annotates a legal document.

3. Script Task: A task that is executed by a computer application or script. It's automated and doesn't require human intervention.
    Examples:
    - Data Backup: An automated script that performs regular data backups in an IT system.
    - Running a Credit Check: An automated process that runs a credit score check when a loan application is submitted.
    - Generating Monthly Reports: A script that automatically compiles and generates monthly financial reports.

4. Manual Task: A task that is performed manually by a human actor without the aid of any business process execution software. An example could be physically moving a document from one place to another.
    Examples:
    - Physical Inventory Check: A warehouse employee manually checks inventory levels.
    - Hand-delivering a Document: Personally delivering a confidential document to a department.
    - Setting up a Conference Room: Arranging chairs and equipment in a conference room for a meeting.

5. Business Rule Task: A task that involves the execution of a business rule. This usually relies on a business rule engine and is automated.
    Examples:
    - Pricing Calculation: Automatically calculating the price of an insurance policy based on a set of business rules.
    - Eligibility Check for a Service: Determining if a customer is eligible for a premium service based on predefined criteria.
    - Discount Application: Applying discounts to a purchase order based on customer loyalty levels.

6. Service Task: A task that uses an automated service, such as a web service or other automated application, to complete its function.
    Examples:
    - Credit Card Transaction Processing: Using an external service to process credit card transactions.
    - Email Notification Service: Automatically sending email notifications through an email service provider.
    - Geolocation Service: Using a geolocation service to find the nearest store location for a customer.

7. Send Task: A task that sends a message to an external participant (outside the process). Once the message is sent, the task is completed.
    Examples:
    - Sending Order Confirmation Email: Automatically sending an email to confirm an order placement.
    - Dispatching Alert to Supplier: Sending an automatic alert to a supplier when stock levels are low.
    - Customer Feedback Request: Sending a feedback request email to a customer after a service.

8. Receive Task: The opposite of a Send Task, this task waits for a message from an external participant. The task is completed when the message is received.
    Examples:
    - Waiting for Customer Payment: A process that waits for a customer's payment confirmation.
    - Receiving Supplier Acknowledgement: A task that waits for an acknowledgment from a supplier after sending a purchase order.
    - Awaiting Regulatory Approval: Waiting for approval from a regulatory body for a new product launch.

9. Call Activity (Sub-Process): This is not a task in the strict sense but is worth mentioning. It represents a call to another process or a reusable subprocess. It can be collapsed (hidden) or expanded (shown) in the diagram.
    Examples:
    - Employee Onboarding Process: Calling a subprocess that handles various steps involved in onboarding a new employee.
    - Order Fulfillment Process: A subprocess that handles all steps from picking, packing, and shipping an order.
    - Customer Complaint Resolution: A subprocess that deals with the steps to address and resolve customer complaints.

Task Markers:

Task Markers provide additional information about the nature of the task without complicating the overall diagram.

1. Loop Marker: Indicates that the task is repeated multiple times. The loop can be based on a condition (conditional loop) or can be executed a fixed number of times (standard loop).
   - Example 1: A "Customer Feedback Follow-up" task in a customer service process might use a loop marker to indicate that the follow-up is repeated until satisfactory feedback is received.
   - Example 2: A "Quality Check" task in a manufacturing process may have a loop marker to represent the repeated inspection of products until they meet quality standards.
   - Example 3: In a sales process, a "Client Contact" task might be looped until a response is received from the client.

2. Multiple Instance Marker: This marker shows that the task will be executed multiple times in parallel or sequentially, creating multiple instances of the task.
   - Example 1: A "Document Review" task in a legal process could be marked to show that multiple legal experts review the document independently at the same time.
   - Example 2: In a recruitment process, "Interview Candidate" tasks might occur simultaneously with different interviewers and candidates.
   - Example 3: "Processing Customer Orders" in an online retail business can occur in parallel, with each order being processed independently.

3. Compensation Marker: Indicates that the task is designed to reverse or compensate for a previous activity in case of exceptions or undo scenarios.
   - Example 1: In a banking process, a "Refund Transaction" task with a compensation marker is used to reverse a transaction in case of customer disputes.
   - Example 2: A "Restore Data" task in an IT process might have this marker to indicate it compensates for a previous task that accidentally deleted data.
   - Example 3: In a travel agency process, a "Cancel Booking" task with a compensation marker is used to reverse a booking in case of customer cancellation.

Subprocesses:

Subprocesses help in simplifying complex process diagrams by encapsulating a detailed sequence of activities within a single activity element.

There are two main types of subprocesses in BPMN:

1. Embedded Subprocess: Directly defined within the main process and cannot exist outside of it. It's depicted as a rounded rectangle with a plus sign.

2. Call Activity: Represents a reusable subprocess that can be defined once and called from multiple processes. It's depicted similarly to an embedded subprocess but represents a reference to an external process.

Examples of subprocesses include:

1. Employee Onboarding Process: This subprocess might include a sequence of tasks such as 'Fill out HR Forms', 'Setup Workstation', and 'Orientation Training'. It's a distinct set of activities that form part of the larger HR management process.

2. Order Fulfillment Process: In a retail or e-commerce business, this subprocess can encapsulate tasks like 'Check Inventory', 'Package Products', and 'Arrange Shipment'. These activities are part of the broader order management process.

3. Customer Complaint Resolution: This subprocess could involve steps like 'Log Complaint', 'Investigate Issue', and 'Resolve and Follow-up with Customer'. It's a specific workflow within a larger customer service process.

Call Activity:
A Call Activity in BPMN (Business Process Model and Notation) is a special type of subprocess used to represent a reusable business process. Unlike an embedded subprocess which is defined within the context of a larger process, a Call Activity refers to a process that is external and defined elsewhere. This makes it a powerful tool for modularizing and reusing complex sequences of activities across various business processes. The Call Activity is depicted as a rounded rectangle, similar to a task, but typically includes a plus sign in the lower center to denote that it's a callable element.

Here are three examples of Call Activity in business processes:

1. Customer Onboarding in a Bank: When a new customer opens an account, the bank might have a standardized onboarding process. This process could include activities like verifying customer identity, setting up account details, and providing initial financial advice. Instead of defining these steps each time, a Call Activity can be used in various account opening processes (like checking, savings, or loan accounts) to invoke the standard customer onboarding process.

2. Order Processing in E-Commerce: An e-commerce platform may use a Call Activity for the order processing phase, which could be a complex process involving inventory check, payment processing, and order confirmation. Since order processing is a common activity across various types of orders (e.g., standard, expedited, international), it can be defined once as a callable process and used in different sales workflows.

3. Annual Employee Review in HR Management: Human Resources might have a standardized process for conducting annual employee reviews, including performance evaluation, feedback sessions, and development planning. This process can be encapsulated in a Call Activity, allowing it to be used across different departments or units within the organization without the need to redefine the steps each time.

Flows:

1. Sequence Flow: 
   - Sequence flows show the order in which activities will be performed in a process. 
   - Examples:
     1. A sequence flow from a "Start Event" to a "Task".
     2. A sequence flow connecting two tasks, indicating the first task must be completed before the second starts.
     3. A sequence flow from a "Decision Gateway" to different tasks, based on the decision made.

2. Message Flow: 
   - Message flows are used to show the flow of messages between two participants (entities/processes) that are prepared to send and receive them.
   - Examples:
     1. A message flow from a "Customer" process to an "Order Processing" process, indicating a customer placing an order.
     2. A message flow between two different organizations, like an order request from a company to a supplier.
     3. A message flow from a "Human Resources" process to a "Payroll" process, carrying information about employee hours worked.

3. Association: 
   - Associations are used to link information, artifacts, or text to flow objects.
   - Examples:
     1. An association linking a "Task" to a "Text Annotation", providing additional information about the task.
     2. An association connecting a "Data Object" to an "Activity", indicating the data required for that activity.
     3. An association between an "Intermediate Event" and a "Document", showing that the event is triggered by the arrival of a specific document.

Events:

1. Start Events: These events indicate where a process will start:
   - None Start Event: A simple start event without any specific trigger, for example, starting a customer onboarding process when a new account is created.
   - Message Start Event: Triggered when a message is received, for example, starting an order fulfillment process when a customer places an order.
   - Timer Start Event: Starts at a specific time or after a specific duration, for example, starting a monthly inventory review process on the first day of each month.

2. Intermediate Events: These events occur between the start and end of a process:
   - Message Intermediate Event: Occurs when a message is sent or received during the process, for example, sending a confirmation email to a customer in the middle of an order processing workflow.
   - Timer Intermediate Event: Occurs at a defined point in time or after a period within the process, for example, a delay in a shipment process that triggers a notification if a package isn't shipped within 48 hours.
   - Error Intermediate Event: Represents an error that occurs during the process, for example, an error in a payment processing system that triggers an exception handling routine.

3. End Events: These signify the end of a process or a path within it.
   - None End Event: A standard end event that signifies the completion of a process, for example, concluding a customer service inquiry once the issue is resolved.
   - Message End Event: Indicates that the process ends with sending a message, for example, sending a delivery confirmation email at the end of a delivery process.
   - Termination End Event: Denotes an abrupt end to the entire process, for example, ending all activities in a project management process if a critical risk is identified.

4. Timer Events: These events represent specific times or durations within a process. They can start a process, delay it, or trigger actions at defined times.
   - Examples:
     - Timer Start Event: Starting a monthly payroll process on the last day of each month.
     - Timer Intermediate Event: Triggering a reminder email if a task is not completed within 24 hours.
     - Timer End Event: Ending a promotional campaign at midnight on a specific date.

5. Message Events: These events represent interactions with external communication parties. They can be used to start, delay, or complete a process based on message exchanges.
   - Examples:
     - Message Start Event: Starting a support ticket process when a customer email is received.
     - Message Intermediate Event: Sending a request for approval and waiting for a response before proceeding.
     - Message End Event: Completing a process by sending a final report to a client.
